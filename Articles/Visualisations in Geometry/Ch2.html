Chapter 2 Visualisation Techniques

2.1 Colour

As we will be producing visualisations to be displayed on a computer monitor, the colours that we have to work with are those which can be produced by a computer monitor. Every monitor will produce colours slightly differently, but the specifics here are not a focus of this project. For our purposes it suffices to think of the gamut (the set of producible colours)[AGO05] of our monitor as the unit cube [0,1]×[0,1]×[0,1]⊂ℝ3 describing the red, green, and blue (RGB) components of a colour. We will denote the set of colours as Col.

Another useful parameterisation of Col is through the HSV (Hue, Saturation, Value) model. As discussed in [AGO05], this way of describing colours arises by looking down the main diagonal of the RGB cube, and parameterising colours in terms of their hue, saturation, and value. The hue of a colour it’s angle around the main diagonal, with the convention that 0 is red, 23⁢π is green, and 43⁢π is blue. For the rest of this document we will assume the hue of a colour to be re-scaled to be in the range [0,1] so that 0,1 are both red, 13 is green, and 23 is blue. The saturation of a colour is its distance from the main diagonal and represents the intensity of the colour, or how much it differs from grey. The value of a colour is how far from black it is. A colour with value 0 is black no matter its hue and saturation, and a colour with value 1 can be white, or any other combination of hue and saturation as long as at least one of the RGB components is 1. Both the saturation and value components are re-scaled to be in the range [0,1].

One important thing to note with HSV components is that not all combinations of h∈[0,1],s∈[0,1],v∈[0,1] produce distinct colours. Instead we have the following:

Remark 2.1.1.

Let C=HSV⁡(hC,sC,vC) and D=HSV⁡(hD,sD,vD) be colours such that C=D. Then one of the following is true:

  • •

    hC=hD and sC=sD and vC=vD,

  • •

    sC=sD=0 and vC=vD,

  • •

    vC=vD=0.

That is to say, if the colour has 0 value, than the other components do not matter, and if the colour has 0 saturation, then the hue component does not matter.

Figure 2.1 shows side-by-side the RGB colour cube, and the HSV colour cylinder. This picture makes remark 2.1.1 clear.

With this perspective on colour, as a subset of ℝ3, we can think of it as a topological space, and consider which maps into Col are continuous for example. This idea is central to the rest of the chapter.

2.2 Domain Colouring

In Farris’s 1998 review [FAR98] of Needham’s book “Visual Complex Analysis” [NEE97], Farris introduces a new way of visualising complex functions of one variable, the technique of domain colouring. Informally, domain colouring works by assigning a colour to each point x in the domain of a function f depending on the output f⁢(x). The value of domain colouring for gaining insight about complex functions is mentioned in the paper [FAR17], where we have quote

“… it is more closely tied to what students have seen in functions of one and two variables: a diagram that shows the output values assigned to inputs in a domain.”

It is certainly desirable that a visualisation of a function allows you to understand the output for each input in an unambiguous way. And using the graphs of functions can be an extremely useful way of doing this. Here we take a short digression to some set theory.

Definition 2.2.1.

[BRI98] Let f:X→Y be a function. Then the graph of f is the subset

Γf={(x,f⁢(x))|x∈X}⊆X×Y.

note we slightly deviate from the original notation in [BRI98] to emphasise the fact that the graph of a function is the section of the projection mapping pr1:X×Y→X;(x,y)↦x. This notation Γf is also used in [LEI24]. We take the following definitions from [LEI24] too:

Definition 2.2.2.

Let X,Y be sets. Then a relation R on X and Y is a subset R⊂X×Y. We say that x∈X and y∈Y are related if (x,y)∈R. In this case, write x⁢R⁢y.

There are are some notable kinds of relations, such as equivalence relations which are used in constructing quotients for example. There is another distinguished class of relations called functional relations. Functional relations get their name because they are relations that encode the information of a function.

Definition 2.2.3.

Let R be a relation on X and Y. If for each x∈X there exists a unique y∈Y such that x⁢R⁢y, then we say that R is a functional relation.

The following theorem from [LEI24] tells us that really, functions and their graphs hold the same information.

Theorem 2.2.1.

Let X and Y be sets.

  • •

    For every f:X→Y, the graph Γf is a functional relation between X and Y.

  • •

    For every functional relation R between X and Y, there is a unique function f:X→Y such that Γf=R.

In summary, to know the graph of a function, is to know the function itself.

Example 2.2.1.

In the case that X=Y=ℝ, graphs are what we would typically think of as the graph of a function, for example, the function f:ℝ→ℝ;x↦x2 has graph {(x,x2)|x∈ℝ}. This elementary example is illustrated in figure 2.2.

Now, similar to [SH15], we make rigorous the idea of a domain colouring. Domain colouring is a way of assigning a colour to each point in the domain of a function, for the purpose of visualisation. Functions with easy-to-visualise domains, such as complex functions of one variable, are particularly well suited to this approach. In order to do domain colouring, we require a way of assigning colours to each point in a set.

Definition 2.2.4.

Let X be a set. Then a colour scheme for X is a map κ:X→Col.

A colour scheme could be described in terms of RGB or HSV components. If X has some extra structure then we may want our colour scheme to somehow preserve this structure. For example, if X is a topological space, then we may want a colour scheme for X to be continuous, or for a colour scheme for a normed vector space we may want to assign brighter colours to vectors with larger norms.

Definition 2.2.5.

Let f:X→Y be a function between sets X and Y and κ be a colour scheme for Y. Then the domain colouring of f with colour scheme κ is the colour scheme for X given by κ∘f:X→Col.

Now, to make use of this digression into set theory and graphs of functions. In the case where our colour scheme κ for Y is injective, the domain colouring of a function f:X→Y can be thought of as the graph of f, where we have

{(x,κ∘f⁢(x))|x∈X}⊆X×κ⁢(Y)⊆X×Col

meaning that it tells us everything about the function f. We can tell exactly where each x∈X maps to in Y by which colour it is assigned because every point in Y is assigned a different colour by κ. We then use the domain colouring of f to visualise f by drawing the domain X (or at least some subset of it) with each point x∈X having colour κ∘f⁢(x). This is in contrast with the usual way of visualising the graph of an ℝ-valued function of 1 or 2 variables, which is done by drawing a 2 or 3-dimensional plot of a curve 2.2 or surface sitting above the domain of the function.

Definition 2.2.6.

Let Y be a set and κ:Y→Col be a colour scheme, Then we call κ faithful if it is injective.

The motivation for definition 2.2.6 is that it ensures that for functions f,f′:X→Y, their domain colouring’s κ∘f and κ∘f′ are equal if and only if f=f′. The use of the word faithful then has a similar meaning to in the context of group actions, or more generally, functors.

We now see an example of a colour scheme κℂ:ℂ→Col for the complex plane given by

κℂ⁢(r⁢ei⁢θ)=HSV⁡(12⁢π⁢θ,min⁡{1,1r},min⁡{1,r}).

This colour scheme is well-defined as r→∞ and has the property that at ∞, in the extended complex plane, that the colour κℂ⁢(∞) is pure white (0 saturation and 1 value). Also, at the origin it is pure black. Being black at the origin, white at ∞, and having the hue be equal to the argument is a common convention for colour schemes for ℂ [SH15]. The colour scheme κℂ is shown for a portion of the complex plane near the origin in (figure 2.3), as well as a domain colouring for a complex function f⁢(z)=z+1+iz−1 using this colour scheme (figure 2.4).

In figure 2.4, we see the pole at z=1 as the white point, and the zero at at −1−i as the black point.

In [PP09] domain colourings are used not only for the complex plane itself, but also for Riemann surfaces and general parameterised surfaces in ℝ3. They also use colour schemes which have some extra features such as radial lines and concentric circles representing level sets of the modulus and argument, which allows information such as conformality to be seen visually. Also by using Riemann surfaces as the domains of their functions, they allow visualisation of multi-valued complex functions such as the square root or logarithm without an arbitrary choice of branch.

2.3 Double Domain Colouring

In this section we provide an extension to the idea of a colour scheme and domain colouring, which particularly in the setting of topology, will allow us visualise a larger class of continuous functions than standard domain colouring.

Sometimes a continuous domain colouring may not be sufficient to properly represent a continuous map, simply because not every topological space admits a continuous faithful domain colouring. If we treat Col as a topological space, it is a contractible 3-dimensional manifold with boundary, homeomorphic to the closed cube I3⊂ℝ3 through the RGB parameterisation. So any continuous colour scheme for a topological space X will only be injective if X can be embedded in I3. For example, if X is a Klein bottle, ℝ⁢P2, or has more than 3 dimensions, then we cannot put a continuous faithful colour scheme on X.

To get around this, we introduce the idea of a double colour scheme, which is a way of assigning a pair of colours to each point in a space rather than just a single colour.

Definition 2.3.1.

Let Colu2 denote the set of unordered pairs of colours. That is Colu2=Col×Col/∼ where ∼ is the equivalence relation given by (c1,c2)∼(c1′,c2′) if and only if (c1,c2)=(c1′,c2′) or (c1,c2)=(c2′,c1′).

We will use the notation [c,c′] to denote the unordered pair of colours c,c′∈Col in order to distinguish it from the notation (c,c′)∈Col×Col for ordered pairs.

In order to visualise an unordered pair of colours [c,c′], we could draw a checkerboard pattern where one set of squares has the colour c and the other has colour c′. See figure 2.5.

Checkerboards are a good way of representing unordered pairs of colours because swapping which tiles have which colour does not really change how a checkerboard looks. whether we have a black and white checkerboard or a white and black one, it still looks more or less the same. Because of this visual indistinguishability between checkerboards with their colours swapped we choose to use unordered pairs of colours for our following generalisation of colour schemes, so that the colour information that can be determined visually is unambiguous near a point.

Definition 2.3.2.

Let X be a set. Then a double colour scheme is a map τ:X→Colu2.

This is in fact a generalisation of the idea of a colour scheme, because if we were to have a double colour scheme τ:X→Colu2 where for each y∈X, τ⁢(x)=[c,c] for some c∈C⁢o⁢l, then this would be equivalent to a colour scheme on X.

Now we can see an example of a double colour scheme being put into practice. Consider the following problem. We want to find a continuous colour scheme κ for the 2-sphere S2 which has the property that for all x∈S2, κ⁢(𝐱)=κ⁢(𝐱′) if and only if 𝐱=±𝐱′. This is not possible, because a continuous colour scheme with this property would be equivalent to an embedding of ℝ⁢P2 (obtained by identifying antipodal points of the 2-sphere) into 𝕀3⊂ℝ3, which is not possible. We can however find a faithful (injective) colour scheme for S2. Take for example κ⁢(𝐱)=RGB⁡(x1+12,x2+12,x3+12), which embeds S2 into the RGB colour cube. We can then also consider the colour scheme κ− given by κ−⁢(𝐱):=κ⁢(−𝐱). Using these two colour schemes we can construct the double colour scheme

[κ,κ−]:S2 →Colu2
𝐱 ↦[κ⁢(𝐱),κ−⁢(𝐱)].

This now is a double colour scheme with the property that [κ,κ−]⁢(𝐱)=[κ,κ−]⁢(𝐱′) if and only if 𝐱=±𝐱′.

We can visualise our double colour scheme by drawing S2, with a checkerboard pattern over it’s surface, and colouring one of the sets of tiles by κ and the other set by κ−. Figure 2.6 shows the colour schemes κ and κ− as well as the double colour scheme [κ,κ−]. In the interactive version, drag with the mouse to rotate the spheres to see that the checkerboard colours on antipodal points are indeed the same.

When producing the image of a 2-sphere on screen using the standard computer graphics pipeline, we can think of the process in a simplified way as a composition of maps

S2→ℝ3→ℝ2

where the first map from S2 to R3 is an embedding of S2 into R3 and the second map, from ℝ3 to ℝ2, is the perspective projection map

(x,y,z)↦(xz,yz).

Then only a rectangular portion of ℝ2, which we identify with I×I is actually drawn to the screen. Each of these spaces, S2,ℝ3,I×I can be given a checkerboard pattern with which we could visualise a double colour scheme. In figure 2.6, the checkerboard pattern is rendered in a way that is intrinsic to S2. In figure 2.7 we see methods of visualising a double colour scheme which use checkerboards coming from ℝ3 and I×I too, compared side-by-side. Axis-aligned great circles are drawn on each sphere to give a visual cue for which side of the sphere is facing the viewer.

This demonstration of different checkerboard patterns highlights the fact that the specific pattern used to visualise a double colour scheme τ:X→Colu2 is not important, only that in some small neighbourhood of each point y∈X both colours from the unordered pair of colours τ⁢(x) are present. Some balance must be stuck between choosing a pattern that is coarse enough that the 2 colours present near any point don’t appear to the eye to be mixed into a single colour, yet fine enough that we still have each colour present near any given point in X.

Just as we use colour schemes to define domain colouring, we use double colour schemes to define double domain colouring:

Definition 2.3.3.

Let f:X→Y be a function and τ:Y→Colu2 be a double colour scheme. Then the double domain colouring of f with double colour scheme τ is the double colour scheme for X given by τ∘f.

Examples of double domain colourings will be seen later in figures 4.2 and4.3 when visualising the 3-sphere being obtained as a quotient of the 3-ball by it’s boundary, and the two-fold covering map S⁢U⁢(2)→S⁢O⁢(3).

2.4 Implementation

All of the 3D graphics seen in this document were produced using JavaScript, running in a web browser. Specifically, the JavaScript library p5.js [19] was used to provide access to basic graphical functions such as setting up a canvas to draw to, spatial transformations such as translations and rotations of the view, and drawing shapes on screen, e.g. points, lines, cubes, circles. In order to have the interactive visualisations run in real time, the graphics are all implemented using WebGL. This allows for much of the graphical processing to be done by the computer’s GPU (Graphical Processing Unit) which is hardware designed for this exact purpose. Using WebGL also allows for computations which must be repeated many times (such as computing the positions of all of the points in Figure 3.4) to be done in parallel through the use of shaders, allowing for real-time interactivity. More information about computer graphics using WebGL be found in [ML13].
