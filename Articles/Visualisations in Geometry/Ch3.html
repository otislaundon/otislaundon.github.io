Chapter 3 Visualising SO⁢(3)

In future chapters, we will be visualising the relationship between SU⁢(2) and SO⁢(3), and also the fundamental group of SO⁢(3). This chapter serves as both a prerequisite for those future chapters, and as a place to apply the visualisation techniques from the previous chapter, in order to gain a better understanding of the structure of SO⁢(3).

3.1 Descriptions of SO⁢(3)

First, let’s present some different ways of thinking about rotations in 3 dimensions, i.e. elements of SO⁢(3). There are several different ways to describe elements of SO⁢(3), each with their own interpretations, strengths, and weaknesses. A good place to start is with the definition. Recall that SO⁢(3) is the set of orthogonal 3×3 real matrices with determinant 1. i.e.

SO⁢(3)={A∈Mat⁢(3×3,ℝ)|A⁢AT=𝕀,detA=1}.

Matrices of this form represent linear transformations that are rotations, and act on column-vectors in ℝ3 by left-multiplication. Recall also that it’s Lie algebra is

𝔰⁢𝔬⁢(3)={A∈Mat⁢(3×3,ℝ)|AT=−A,tr⁢(A)=0},

the set of skew-symmetric traceless 3×3 real matrices.

Matrices are a convenient way of representing rotations when it comes to computing things. However, they are wasteful. That is, 3×3 matrices use 32=9 numbers to represent a linear transformation, but SO⁢(3) only has dimension 3. If we are only interested in rotations, then much of this information is redundant. Some ways of describing elements of SO⁢(3) other than by their matrices are by

  • •

    Angle-axis,

  • •

    Euler-angles,

  • •

    Unit-length quaternions,

  • •

    Möbius transformations ℂ→ℂ (of a certain kind).

The angle-axis and Euler-angles parameterisations of SO⁢(3) give charts that are dense in SO⁢(3). The methods using quaternions and Möbius transformations each describe rotations using their own group structure, as well as a homomorphism to SO⁢(3). Below we give more details about each of these descriptions of rotations.

3.1.1 Angle-axis

Here we introduce the theory that formalises the intuitive fact that every rotation in ℝ3 is a rotation about some axis by some angle. In the process, we will also gain some topological insight about SO⁢(3), as well as a way of visualising SO⁢(3). The steps taken in this section closely follow Appendix A of [NAB10].

By the Euler rotation theorem [CON26], every rotation in 3 dimensions other than the identity fixes a unique axis. We call this the axis of a rotation.

Let θ∈ℝ and 𝐮=(xyz)∈ℝ3 be a vector of length 1. Start by defining the map ϕ:ℝ3→𝔰⁢𝔬⁢(3) by

ϕ⁢(𝐮)=(0−zyz0−x−yx0). (3.1)

Let U=ϕ⁢(𝐮). Then we have that

U2=(−(y2+z2)x⁢yx⁢zx⁢y−(x2+z2)y⁢zx⁢zy⁢z−(x2+y2))

and U3=−U,U4=−U2,U5=U and so on such that the odd powers of U are U2⁢n+1=(−1)n⁢U for n≥0 and the even powers are U2⁢n=(−1)n+1⁢U2 for n≥1. Therefore, when we take the matrix exponential

exp⁡(θ⁢U)=∑n=0∞1n!⁢(θ⁢U)n

we can split up the sum into odd and even terms, which gives

𝕀+∑n=0∞1(2⁢n+1)!⁢(θ⁢U)2⁢n+1+∑n=1∞1(2⁢n)!⁢(θ⁢U)2⁢n
= 𝕀+U⁢∑n=0∞(−1)n(2⁢n+1)!⁢θ2⁢n+1+U2⁢∑n=1∞(−1)n+1(2⁢n)!⁢θ2⁢n
= 𝕀+U⁢∑n=0∞(−1)n(2⁢n+1)!⁢θ2⁢n+1+U2⁢(1+∑n=0∞(−1)n+1(2⁢n)!⁢θ2⁢n)
= 𝕀+sin⁡(θ)⁢U+(1−cos⁡θ)⁢U2. (3.2)

where 𝕀 is the 3×3 identity matrix.

It turns out that this is the rotation matrix which describes the rotation about axis 𝐮 by angle θ.

What we have done is gone from ℝ3 to 𝔰⁢𝔬⁢(3), the set of skew-symmetric real 3×3 matrices, and then to SO⁢(3). Writing down the cross matrix θ⁢U for vector θ⁢𝐮 is identifying ℝ3 with 𝔰⁢𝔬⁢(3) by mapping θ⁢𝐮=θ⁢(uxuyuz) to the element θ⁢(ux⁢Ux+uy⁢Uy+uz⁢Uz)∈𝔰⁢𝔬⁢(3) where

Ux=(00000−1010),Uy=(001000−100),Uz=(0−10100000). (3.3)

Then, using the matrix exponential map, which we know is the same as the Lie algebra exponential map in the case of matrix groups, we get an element of SO⁢(3). This gives us a nice link between the angle axis forms of rotations and Lie theory. The exponential map from 𝔰⁢𝔬⁢(3) to SO⁢(3) has the following properties:

Lemma 3.1.1.

[NAB10] For any θ∈ℝ and 𝐮 of unit length, we see that exp⁡(θ⁢𝐮) is in SO⁢(3). Conversely, given a rotation R in SO⁢(3), there exists a unique angle θ∈ℝ satisfying 0≤θ≤π and a unit vector 𝐮 such that R=exp⁡(θ⁢𝐮). Furthermore,

  1. 1.

    If 0<θ<π then 𝐮 is unique,

  2. 2.

    If θ=0 then R=exp⁡(θ⁢𝐮) for any unit vector 𝐮∈ℝ3

  3. 3.

    If θ=π, then R=exp⁡(θ⁢𝐯) iff 𝐯=±𝐮.

From lemma 3.1.1 we can see that every point in SO⁢(3) is exp⁡(θ⁢𝐮) for some θ⁢𝐮∈B0⁢(π)¯, so exp restricted to a closed ball of radius π is surjective. Going the other way, from SO⁢(3) to B0⁢(π)¯⊂𝔰⁢𝔬⁢(3), is the parameterisation by logarithmic coordinates. We know from general theory that logarithmic coordinates for Lie groups are well defined in a neighbourhood of the origin. Indeed, we can see from 3.1.1 they are defined on the neighbourhood of 𝕀∈SO⁢(3) that maps to the interior of a ball of radius π in 𝔰⁢𝔬⁢(3), which is a maximal dense chart.

Since the map exp is a continuous map (which is clear from 3.2), the universal property of quotients gives us a continuous bijection B0⁢(π)¯/∼→SO(3) where ∼ is the equaliser of exp. The equaliser ∼ of exp is the equivalence relation given by θ⁢𝐮∼θ′⁢𝐱′ if and only if exp⁡(θ⁢𝐮)=exp⁡(θ′⁢𝐮′), which by 3.1.1 is true iff θ=θ′=π and 𝐮=±𝐮′, and so it is the equivalence relation that identifies antipodal points of the boundary sphere. The quotient space B0⁢(π)¯/∼ is homeomorphic to ℝ⁢P3 [HAT02]. So we have a continuous bijection ℝ⁢P3→SO⁢(3), and since B0⁢(π)¯/∼ is compact and SO⁢(3)⊂GL⁢(n,ℝ)⊂ℝn2 is Hausdorff, this map is a homeomorphism by Theorem 3.7 from [ARM83]. Therefore, we have that SO⁢(3) is homeomorphic to ℝ⁢P3. This tells us that the fundamental group of SO⁢(3) is ℤ/2⁢ℤ [NAB10], a fact that we will demonstrate in chapter 4.3.

This angle-axis description of SO⁢(3) will be particularly useful for visualisation, as we can easily draw points in a ball of radius π, the interior of which is a maximal dense chart for SO⁢(3). The main benefit with this technique is that a rotation can be visualised as a single point on screen, rather than needing to show it’s action on some object. This opens up more possibilities for visualising collections of rotations and relationships between them. The main limitation is that the boundary points, corresponding to rotations by π radians about some axis, will not be properly visualised since they will not be represented by a unique point in the visualisation. The viewer must therefore remember that there should be this identification of antipodal points happening on the boundary.

One idea for visualising the identification of antipodal points would be to assign a continuous colour scheme to the 2-sphere in such a way that two points 𝐱,𝐱′ have the same colour if and only if x=±x′. Unfortunately, this is not possible as discussed in 2.3 since such a colour scheme would be equivalent to embedding ℝ⁢P2 into ℝ3. But since there is no injective continuous map ℝ⁢P2→ℝ3, there would necessarily be a pair of non-antipodal points with the same colour.

But recall, we have already described a solution using double colour schemes to this visualisation problem and demonstrated it in Figure 2.6. It is possible to pick an unordered pair of colours at each point on the sphere in such a way that we pick the same pair of colours for antipodal points, and different pairs of colours for each pair of non-antipodal points. And crucially, this can be done in a continuous way.

In Figure 3.1, we pick an element of SO⁢(3) and denote it by θ⁢𝐮 using the angle-axis notation. We then visualise the point θ⁢𝐮 inside the ball of radius π with antipodal points identified (identification shown by double colour scheme [κ,κ−]) on the left hand side of Figure 3.1. Alongside the point θ⁢𝐮 itself, we show the action of θ⁢𝐮 on ℝ3. This is done by showing a cube that is rotated around 𝐮 by angle θ relative to a set of fixed axes. The cube has faces coloured by their normal direction, using the convention that x is red, y is green, and z is blue. For example, the face of the cube that is originally oriented in the positive z direction is blue, and the face in the negative z direction would be yellow, the complimentary colour of blue.

In the interactive version of Figure 3.1, the user can drag with the mouse on the cube on the right hand side of the visualisation to rotate it, thus changing both θ and 𝐮. You can also drag with the mouse on the left hand side of Figure 3.1 to rotate the whole visualisation in order to see it from a different perspective. Pay attention to what happens when θ jumps discontinuously from π to −π (or vice versa). On the left hand side of Figure 3.1 the point θ⁢𝐮 will move suddenly, but on the right hand side we see that this is only an artefact of using a ball with antipodal points identified for the visualisation, as the rotation around 𝐮 by angle θ does not suddenly change as θ goes from π to −π.

We can also extend our double colour scheme from the surface of the ball to the whole ball itself. As it is currently defined, for a point (xyz)∈S2⊂ℝ3, we have colour scheme κ:S2→Col given by RGB⁡(x+12,y+12,z+12). But this formula also defines a faithful colour scheme on B0⁢(1)¯ (which can be scaled up by a factor of π to match our visualisation of SO⁢(3)). We can therefore use κ as before to define a double colour scheme τ:B0⁢(1)¯→Colu2 by

τ⁢(𝐱)=[κ⁢(𝐱),r⁢κ⁢(−𝐱)+(1−r)⁢κ⁢(𝐱)] (3.4)

where r=|x|. Note that when restricted to the boundary of B0⁢(1)¯ it coincides with [κ,κ−]. It is not that obvious that the double colour scheme τ is faithful on the interior, but it can be shown to be due to the asymmetry of how colours κ⁢(𝐱) and κ⁢(−𝐱) are mixed. Rather than using τ as is, which is grey at the centre of B0⁢(1), we attenuate τ to black towards the centre by multiplying the RGB components of each of the colours in τ by r=‖𝐱‖. Define τ0 to be

τ0⁢(𝐱)=[r⁢κ⁢(𝐱),r2⁢κ⁢(−𝐱)+(r−r2)⁢κ⁢(𝐱)]. (3.5)

This then means that darker colours correspond to elements closer to the identity, thus associating the brightness of the colours with the magnitude of the rotations. It also retains the feature that on the surface of the ball, distinct points have the same colour if and only if they are antipodal. When using τ0 to visualise a map into SO⁢(3), it will be clear which points are mapping near to the identity, and which map to rotations by angle π. Also by looking at the hue of the colours, the axis of rotation can be determined. Continuous maps will also have continuous double domain colourings if they use τ0 as their double colour scheme.

In Figure 3.2 we visualise the double colour scheme τ0 for the unit ball. The black great circle on the left hand side of Figure 3.2 shows the cross section (which, with antipodal boundary points identified is homeomorphic to ℝ⁢P2) of B0⁢(1)¯ for which τ0 is visualised on the right hand side of 3.2. By taking cross-sections we get to see the double colour scheme applied to any part of the interior of the ball, which we cannot see from the outside. In the interactive version, drag with the mouse on the left hand side to rotate the view, and drag with the mouse on the right hand side to change which cross section is visualised.

Note that we have not proved that τ0 is faithful. A task for future work is either showing that it is faithful, or constructing a continuous faithful double colour scheme for SO⁢(3) such that it takes value [black,black] at the origin.

3.1.2 Euler Angles

One prevalent way of representing rotations is through Euler angles, which describe a rotation R in ℝ3 by a composite of 3 axis-aligned rotations. There are many different conventions used for exactly which angles around which axes we compose in which order. To avoid any confusion, the convention we will use here is that any rotation can be described as a composite of rotations R=X⁢(Rx)⁢Y⁢(Ry)⁢Z⁢(Rz), where

X⁢(Rx) =(cos⁡Rx−sin⁡Rx0sin⁡Rxcos⁡Rx0000),
Y⁢(Ry) =(cos⁡Ry0sin⁡Ry000−sin⁡Ry0cos⁡Ry),
Z⁢(Rz) =(0000cos⁡Rz−sin⁡Rz0sin⁡Rzcos⁡Rz).

The rotation R is then a rotation in the z axis, followed by a rotation in the y axis, followed by a rotation in the x axis.

We will refer to this as the Euler angles representation of a rotation, but the reader should be aware that this is a Tait-Bryan angles version rather than a proper Euler angles one [CON26].

Euler angles provide a maximal chart for SO⁢(3) if we restrict to the set

{(Rx,Ry,Rz)∈ℝ3|0≤Rx<π,0≤Ry<π,0≤Rz<2⁢π}

then the Euler-angles representation of a rotation is unique. This gives us a way of thinking about rotations as a cube rather than a sphere. We will prefer using the angle-axis representation for our visualisations of SO⁢(3) due to it’s rotational symmetry, and fewer choices (e.g. convention of order of composition) being required.

3.1.3 Möbius Transformations

As in [JS87], let Σ=ℂ∪{∞} denote the extended complex plane, and identify it with the 2-sphere via stereographic projection. Let the unit sphere S2={(x,y,z)∈ℝ3|x2+y2+z2=1}. Then the stereographic projection map π:S2→Σ has formula

π⁢(x,y,z)=x+i⁢y1−z

where in the case z=1, the output should be interpreted as ∞∈Σ.

Definition 3.1.1.

Let f:Σ→Σ. If f is meromorphic and bijective, then we say f is an automorphism of Σ. Denote the group of automorphisms of Σ by Aut⁡(Σ).

Möbius transformations are rational complex functions ℂ→ℂ of the form

z↦a⁢z+bc⁢z+d

where z,a,b,c,d∈ℂ such that a⁢d−b⁢c≠0. In [JS87], the automorphisms of the Riemann sphere are shown to be exactly the Möbius transformations.

There is a surjective group homomorphism GL⁢(2,ℂ)→Aut⁡(Σ) given by

(abcd)↦a⁢z+bc⁢z+d

which has kernel {λ⁢I∈GL⁢(2,ℂ)|λ∈ℂ} [JS87]. It is then a theorem that the Möbius transformations Aut⁡(Σ) form a group isomorphic to the Lie group PGL⁢(2,ℂ) [JS87].

Rotations in ℝ3 (elements of SO⁢(3)) are uniquely determined by their action on S2.

Definition 3.1.2.

[JS87] We define Rot⁡(Σ) to be the set of maps Σ→Σ that are of the form π−1⁢R⁢π where R:S2→S2 is a rotation of the Riemann sphere.

So we have that there is a subgroup Rot⁡(Σ)⊂Aut⁡(Σ) which is isomorphic to SO⁢(3). We then have the theorem

Theorem 3.1.1.

[JS87]

Rot(Σ)={T:z↦a⁢z+b−b¯⁢z+a¯||a|2+|b|2=1}=PSU(2,ℂ).

Using the correspondence between Möbius transformations and 2×2 complex matrices, we get that any matrices of form

(ab−b¯a¯)

define rotations, and moreover two matrices A,B of this form define the same rotation if and only if A=−B which we can see from the fact that

a⁢z+b−b¯⁢z+z¯=−z−bb¯⁢z−z¯.

These matrices are exactly those corresponding to elements of SU⁢(2), or equivalently, unit quaternions.

The way in which Möbius transformations arise from rotations of the Riemann sphere is visualised in Figure 3.3. The complex plane has the domain colouring (with colour scheme κℂ) of the Möbius transformation applied to it which arises from a rotation of the Riemann sphere. The Riemann sphere is also coloured using the domain colouring corresponding to the rotation being applied. The original colour scheme on the Riemann sphere is compatible with that of the plane under stereographic projection. It is given by κℂ∘π:Σ→Col. In the interactive version, drag the mouse to rotate the Riemann sphere and see how the Möbius transformation corresponding to it’s orientation changes. It should be noted that whilst the sphere rotates one way (shown by the orientation of the wire frame around the sphere), the domain colouring appears to rotate the other way. This is no accident, and is a general fact to keep in mind when looking at domain colourings. The motion of the points themselves is inverse to the motion of the colours. This is analogous to the linear equation (x−1) having it’s root at 1, not at −1, which can feel counter intuitive when first learning about linear equations. The root is the point that maps to 0, not the point where 0 maps to.

3.2 Left, Right, and Conjugation Actions on SO⁢(3)

Now that we have a way of viewing many points simultaneously in SO⁢(3), by drawing all of their angle axis representations, we can visualise:

  • •

    Conjugation-action by an element.

If we fix some initial configuration of points {ai}1≤i≤k in SO⁢(3) as well as one distinguished point b, then we can visualise the left action Lb on SO⁢(3) by drawing Lb⁢ai=b⁢ai for each 1≤i≤k for different values of b. In Figure 3.4 we see just that. The initial configuration of points is a set of randomly chosen positions on spheres of radius π4,π2,3⁢π4, and π. The points Lb⁢ai,1≤i≤k are each marked with a checkered dot. The checker pattern on each dot Lb⁢ai has colours determined by τ0⁢(ai), the double colour scheme for SO⁢(3) that was defined in 3.5. Similar to Figure 3.1, in the interactive version of 3.4 the viewer can drag with the mouse on the right hand side to change the element b and see the effect of map Lb:SO⁢(3)→SO⁢(3) on the points ai,1≤i≤k change on the left hand side.

By drawing the image of each ai under Lb as a dot which retains the original pair of colours τ0⁢(ai), what we see in Figure 3.4 is essentially a double domain colouring for (Lb)−1=Lb−1, because at any given point b⁢ai in SO⁢(3), we see the colour τ0⁢(ai)=τ0∘(Lb)−1⁢(b⁢ai). We can also draw the double domain colouring for Lb itself. This is shown in Figure 3.5, where on the left we see the pair of colours τ0∘Lb⁢(𝐱) for each point 𝐱∈SO⁢(3).

The point b−1 is labelled in Figure 3.5 to highlight the fact that this is the point that gets mapped to 𝟎 (the identity 𝕀∈SO⁢(3)), which corresponds to the fact that the darkest points of the double domain colouring are those around b−1 . We see that when using a fixed checkerboard pattern to show the double domain colouring τ0∘Lb, a visual discontinuity is present when b≠𝕀.

When we show the double domain colouring for the identity map, as in 3.2, there is no issue because the checkerboard pattern is defined in a way that is consistent throughout the interior of the sphere. However, when showing the double domain colouring for the left-action map by a non-identity element the boundary of the ball is mapped into the interior. While the unordered pair of colours immediately on either side of the boundary is the same, which one is assigned to which checker square is swapped. So there is no discontinuity in the double domain colouring itself, but only in the way it is visualised here. At the core of this problem is the fact that an ordered pair of colours uniquely defines an unordered pair of colours by simply “forgetting” their order. But there is no canonical way to go from an an unordered pair to an unordered pair. A choice has to be made. By using a checkerboard to represent an unordered pair of colours, such a choice must be made globally, but we already know that there is no injective continuous map ℝ⁢P3→Col, so on each set of checkers (the “black” and the “white” squares), the colour map will be discontinuous.

A further modification can be made to visualisation of the double domain colouring so that we pull back not only the pair of colours, but also the checkerboard itself, by the map that we are visualising. This is what is shown in Figure 3.6.

Of course we can visualise the right action Rb⁢ai=ai⁢b in the exact same way as we did the left action Lb. Figure 3.7 shows the right action Rb on a similarly chosen initial of set of points.

And once again, in the same way, Figure 3.8 shows the conjugation action Cb applied to an initial set of points.

By interacting with the visualisations 3.4, 3.7, and 3.8, we visually verify some facts. The the Left and Right actions Lb,Rb have the same effect on the one-parameter subgroup generated by b (which in each visualisation 3.4, 3.7, 3.8 is the line through the origin and b), however points not in the one parameter subgroup generated by b are rotated around this line in opposite directions. We also see that Cb fixes the identity for each b as expected, but furthermore we see that the action of Cb is equivalent to applying a rotation to 𝔰⁢𝔬⁢(3), something that was not immediately apparent from the definition. The conjugation action by b actually induces a linear map 𝔰⁢𝔬⁢(3)→𝔰⁢𝔬⁢(3) which is, with respect to the basis Ux,Uy,Uz, the same as the rotation b itself acting on ℝ3. This corresponds to the fact that both the standard and adjoint representations of SO⁢(3) are 3 real dimensional representations which are isomorphic [HAL15].

This can be shown through explicit matrix calculation and some basic results from representation theory.

The standard Lie group representation, which maps A∈SO⁢(3) to the linear map ℝ3→ℝ3 which it defines, has associated Lie algebra representation which takes X∈𝔰⁢𝔬⁢(3) to the linear map that it defines. The adjoint representation Ad:SO⁢(3)→GL⁢(𝔰⁢𝔬⁢(3)) takes the matrix A∈SO⁢(3) to the adjoint map

AdA:𝔰⁢𝔬⁢(3) →𝔰⁢𝔬⁢(3)
X ↦A⁢X⁢A−1

and has associated Lie algebra representation ad:𝔰⁢𝔬⁢(3)→GL⁢(𝔰⁢𝔬⁢(3)) which maps X∈𝔰⁢𝔬⁢(3) to adX, which is the map 𝔰⁢𝔬⁢(3)→𝔰⁢𝔬⁢(3) given by

adX:𝔰⁢𝔬⁢(3) →𝔰⁢𝔬⁢(3)
Y ↦X⁢Y−Y⁢X.

So it suffices to show that ϕ defined in 3.1 satisfies

ϕ⁢(X⁢𝐯)=adX⁢ϕ⁢(𝐯)

for all

X=(0−cbc0−a−ba0)∈𝔰⁢𝔬⁢(3)

and 𝐯=(xyz)∈ℝ3. We have that

ϕ⁢(X⁢𝐯) =ϕ⁢(−c⁢y+b⁢zc⁢x−a⁢z−b⁢x+a⁢y)=(0b⁢x−a⁢yc⁢x−a⁢z−b⁢x+a⁢y0c⁢y−b⁢z−c⁢x+a⁢z−c⁢y+b⁢z0)
=(−c⁢z−v⁢yb⁢xc⁢xa⁢y−c⁢z−a⁢xc⁢ya⁢zb⁢z−b⁢y−a⁢x)−(−c⁢z−b⁢ya⁢ya⁢zb⁢x−c⁢z−a⁢xb⁢zc⁢xc⁢y−b⁢y−a⁢x)
=X⁢ϕ⁢(𝐯)−ϕ⁢(𝐯)⁢X
=adX⁢(ϕ⁢(𝐯)).

Thus, as their associated Lie algebra representations are isomorphic, by proposition 6.1.3, the standard and adjoint representations of SO⁢(3) are isomorphic.
